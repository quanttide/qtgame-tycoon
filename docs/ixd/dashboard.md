## 二、Header 核心指标栏

### 2.1 结构

| 区域 | 内容 |
|---|---|
| 左侧 | Logo + 公司名 |
| 右侧 | 6 个指标项 horizontally |

### 2.2 指标项

| 指标 | 格式 | 颜色语义 |
|---|---|---|
| 💰 现金 | `¥{number}` | 正常: `#f0f6fc` / 低于 30,000: `#f85149` |
| 📈 季度营收 | `¥{number}` | 正: `#3fb950` / 负: `#f85149` |
| ⭐ 信誉 | `{0-100}` | ≥70: `#3fb950` / 40-69: `#d29922` / <40: `#f85149` |
| 👥 团队 | `{正式}+{实习}` | — |
| ⚡ 效率 | `{n}x` | — |
| 📅 季度 | `Q{1-∞}` | — |

### 2.3 状态

- **加载中**: 所有数值显示 `—`，渐入动画 (opacity 0→1, 300ms)
- **数据更新**: 数值变化时闪烁 200ms (background 闪白 → 恢复)
- **临界警告**: 现金低于阈值时，数字左侧出现 ⚠ 红色脉冲点

```

└──────────────────────────────────────┘
```

### 3.3 状态

| 状态 | 处理 |
|---|---|
| 空状态(无项目) | 显示 `暂无项目机会` + 灰色虚线框 + 提示"等待市场刷新" |
| 容量满 | 待承接卡片上的"接"按钮变灰禁用，tooltip "项目容量已满" |
| 可承接 0 / 容量满 | Header badge 变为红色 |
| 执行中为空 | 显示 `暂无执行中的项目` + 提示"去接项目" |
| 项目延期 | 进度超过工期 80% 仍未完成 → 卡片左侧色条变为红色，显示 ⚠ 标记 |
| 新增项目 | 卡片从上方滑入 (translateY(-10px) → 0, 200ms ease-out) |
| 完成项目 | 卡片绿色闪烁 → 从列表消失 (缩放至 0, 200ms ease-in) |

### 3.4 微交互

- 项目完成后资金流入：`+¥15,000` 浮动文字从项目位置向上飘散（1s 后消失）
- 每种知识获取时右侧进度条随之增长（0.3s 过渡）

## 四、团队与人才面板

### 4.1 成员行

```
┌──────────────────────────────────────────┐
│ (头像) 陈小明        📊 85  📚 32  🟢空闲 │
└──────────────────────────────────────────┘
```

| 属性 | 规格 |
|---|---|
| 行高 | 44px |
| 头像 | 32x32 圆形，根据主技能自动选择 emoji (数据>教育=👩‍💻，教育>数据=👨‍🏫) |
| 技能值 | 彩色文字 `#58a6ff`(数据) / `#3fb950`(教育) |
| 状态标签 | 空闲=绿色 `#3fb950`，执行中=黄色 `#d29922` |

### 4.2 实习生区

- 与正式成员视觉区分：头像用 🧑‍🎓，行背景略透明
- 来源高校显示在右侧标签 `浙理工 · 杭电`
- 提示人才缺口：显示 `⚠ 缺 1 名数据方向`

### 4.3 底部统计条

```
🧑‍🎓 实习生: 2 人 (来自: 浙理工 · 杭电)  ⚠ 缺1名数据方向
```

### 4.4 状态

| 状态 | 处理 |
|---|---|
| 空(无团队) | 不可能，初始 3 人 |
| 成员忙碌 100% | 所有状态标签都是黄色，面板底部显示警示条 |
| 人才缺口 | 底部 `⚠` 信息，缺口方向加粗 |
| 新成员加入 | 行从右侧滑入，头像放大弹跳 (scale 0.8→1.1→1.0) |
| 成员离职 | 行变红 → 缩起消失 (300ms) |

### 4.5 操作

- 点击成员行：弹出操作菜单（加薪 / 辞退 / 查看信息）
- 底部"招聘"按钮：点击弹出招聘 modal

### 5.1 布局

```
┌──────────────────────────────────────┐
│ 🔄 产教融合循环                       │
│                                       │
│      Lv.3                             │
│      融合度                            │
│                                       │
│ 📊 ▓▓▓▓▓▓▓░░░░░░░░░  72pts           │
│ 📚 ▓▓▓▓▓▓░░░░░░░░░░  68pts           │

```

### 5.2 融合度等级

| 等级 | 阈值 | 描述文字 | 加成 |
|---|---|---|---|
| Lv.0 | 0 | 同时积累数据与教育知识，激活产教融合循环 | — |
| Lv.1 | max(d,e) ≥ 20 | 产教融合初显成效 | 单价 +10%, 全局效率 +5% |
| Lv.2 | max(d,e) ≥ 40 | 正向循环运转良好 | 单价 +20%, 全局效率 +10% |
| Lv.3 | max(d,e) ≥ 60 | 技术与教育相互赋能 | 单价 +30%, 全局效率 +15% |
| Lv.4 | max(d,e) ≥ 80 | 高度融合！知识资产驱动增长 | 单价 +40%, 全局效率 +20% |
| Lv.5 | max(d,e) ≥ 100 | 产教融合典范！持续创新引擎 | 单价 +50%, 全局效率 +25% |

### 5.3 状态

| 状态 | 处理 |
|---|---|
| 升级 | 等级数字翻转动画 (旧数字向上翻出，新数字从下方翻入，300ms) |
| 知识变化 | 进度条缓动填充 (0.5s ease-out) |
| 空(无知识) | 两条进度条均为 0%，描述显示"等待激活" |
| 最大值 | 进度条填满后出现 ✦ 闪烁装饰 |

### 5.4 关联反馈

- 融合度升级时：全屏中央 toast `🔄 产教融合循环升级 Lv.N！全局效率+N0%` (2s)
- 升级瞬间项目面板中的单价数字闪烁

## 六、战略投资面板

### 6.1 折叠交互

- 默认收起，只显示标题栏
- 点击标题栏展开/收起，箭头旋转 ▲→▼ (200ms)
- 展开时内容区以高度动画展开 (max-height 0→600px, 300ms)

### 6.2 投资项卡片

```
┌──────────────────────────────────────┐
│ 🛠️  数据工具链      ● ● ● ○ ○      │
│      数据项目效率+20%/级  ↑ ¥68,000 │
└──────────────────────────────────────┘
```

| 属性 | 规格 |
|---|---|
| 高度 | 52px |
| 每行 3 项（桌面） / 2 项（平板） / 1 项（手机） |
| 圆点 (dot) | 10px 圆，已点亮=蓝色+发光，未点亮=灰色边框 |
| 升级按钮 | 右侧，显示价格或"已满级" |

### 6.3 按钮状态

| 状态 | 样式 | 交互 |
|---|---|---|
| 可升级 | 金色渐变背景 | hover 缩放 1.05 |
| 不可升级(缺钱) | 灰色背景 | 点击 toast "💰 资金不足" |

| 升级中(不影响) | 添加一个 1s 的脉冲动画表示刚刚购买 | — |

- 购买成功时：货币数额扣减动画（数字逐格减少，300ms）
- toast 提示：`⬆️ 数据工具链 Lv.2`
- 对应效率指标即时刷新

## 九、事件决策模态

### 9.1 结构

```
┌──────────────────────────────────────┐
│            (半透明模糊背景)            │
│  ┌────────────────────────────────┐  │
│  │ 📢 客户变更需求                 │  │
│  │                                │  │
│  │ 清华项目执行过半，客户提出新增  │  │
│  │ 两个分析维度...                │  │
│  │                                │  │
│  │ ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐ │  │
│  │ │ ✅ 接受变更，保证口碑       │ │  │
│  │ │    信誉 +5，工期 +30%      │ │  │
│  │ └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘ │  │
│  │ ┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐ │  │
│  │ │ ❌ 按合同拒绝             │ │  │
│  │ │    信誉 -3，工期不变      │ │  │
│  │ └ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘ │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### 9.2 行为

| 动作 | 效果 |
|---|---|
| 弹出 | 背景 fadeIn (300ms)，卡片 scale 0.85→1.0 + 弹跳 (300ms ease) |
| 点击选项 | 选项背景变色 → 卡片缩小淡出 (150ms) → 移除 |
| 选择后 | 即时显示结果 toast (1.5s) |
| 连续事件 | 有 3 回合冷却 (eventCooldown)，期间不触发新事件 |

### 9.3 响应式

- 移动端：modal 宽度 90%，最大 360px
- 桌面：modal 宽度 400px
- 选项按钮在全屏都撑满宽度

## 十、全局组件 & 微交互

### 10.1 Toast

| 属性 | 值 |
|---|---|
| 位置 | 屏幕中央 (fixed 50%/50%, translate -50%) |
| 背景 | `rgba(0,0,0,.85)` |
| 圆角 | 14px |
| 默认时长 | 2000ms |
| 进入动画 | scale 0.8→1.0 + opacity 0→1 (200ms) |
| 离开动画 | opacity 1→0 (200ms) |
| 多 toast 叠加 | 后一个覆盖前一个（clearTimeout 实现） |

### 10.2 浮动文字 (FloatText)

| 属性 | 值 |
|---|---|
| 位置 | 项目卡片上方随机偏移 |
| 动画 | translateY(0→-50px) + opacity(1→0) |
| 时长 | 1s |
| 触发时机 | 项目完成 +¥、实习生产生成本 |

### 10.3 加载骨架屏

全局数据初始化时（`game.start()` 到首次 `updateUI()`）：

- Header 数字显示 `—`
- 项目面板显示 2 个灰色矩形脉冲条
- 团队面板显示 3 个灰色行脉冲条
- 所有交互按钮禁用
- 持续约 800ms 后渐入真实内容

### 10.4 空状态汇总

| 组件 | 空状态文案 |
|---|---|
| 项目列表(待承接) | "暂无项目机会，等待市场刷新…" |
| 项目列表(执行中) | "暂无执行中的项目，去接一个吧" |
| 团队面板 | 初始 3 人，不存在空状态 |
| 经营趋势图 | "数据不足，继续经营积累趋势" |
| 事件历史 | "暂无事件记录" |

### 10.5 错误边界

- 全局异常捕获：`window.onerror` 输出到 console，页面不崩溃
- 数据校验：`money`、`reputation`、`loopLevel` 等核心值保持 ≥0 和 ≤max
- localStorage 存档读取失败：提示"存档损坏"并清空回默认状态（未来功能）

## 十二、设计资产

| 资产 | 规格 | 用途 |
|---|---|---|
| 圆角 | 12px (面板) / 8px (卡片) / 6px (标签) | 统一 |
| 字体 | `-apple-system, 'Segoe UI', 'PingFang SC', sans-serif` | 系统原生 |
| 字号阶梯 | 20(大字号) / 14(标题) / 12(正文) / 10(辅助) | 层级 |
| 颜色 | 深色背景 `#0a0e17` / 卡片 `rgba(255,255,255,.04)` / 边框 `rgba(255,255,255,.08)` | 大屏暗色调 |
| 动画 | 200-300ms ease-out / ease | 所有过渡 |
| 阴影 | `box-shadow` 仅用于 hover 提亮，不用于深度层次 | 扁平风格 |
