# 量潮科技模拟器 · 产品需求文档

---

## 一、产品概述

### 1.1 一句话定义

以 CEO 视角经营一家产教融合的科技公司，目标是**建一套离开你也能自然运转的制度系统**。

### 1.2 核心哲学

> 不是玩家退休后游戏结束，而是退休后游戏才刚刚开始。

这个设计的原点来自背景中真实公司的经历：创始人从"专制君主"向"立宪君主"、公司从"人治"向"法治"转变。游戏不应该鼓励玩家事必躬亲，而应该鼓励玩家**建造系统**——然后放手。

### 1.3 目标用户

- 对创业/企业管理感兴趣的玩家
- 喜欢策略经营类游戏（如《金融帝国》《软件公司》等）的玩家
- 认同"制度比人重要"管理理念的人

## 二、玩家旅程

### 序幕：从零到一（Q1-Q2）

| 目标 | 教学 |
|---|---|
| 理解基本操作 | 引导教程：如何接项目、看数据 |
| 实现收支平衡 | 完成第一个项目，回笼资金 |
| 认识产教融合 | 同时接触数据和教育两类项目 |

### 第一幕：打基础（Q2-Q4）

| 目标 | 系统解锁 |
|---|---|
| 稳定现金流 | 战略投资面板开放 |
| 招聘第二名成员 | 团队管理功能深化 |
| 激活融合循环 Lv.1 | 产教融合仪表显示 |

### 第二幕：建系统（Q4-Q8）

| 目标 | 系统解锁 |
|---|---|
| 制度成熟度系统上线 | 标准化、去中心化、透明化三维度 |
| 自动化升级见效 | 容量+1，部分决策可委派 |
| 融合循环 Lv.3+ | 获得明显的效率加成 |

### 第三幕：放手（Q8-Q12）

| 目标 | 系统解锁 |
|---|---|
| 制度成熟度 70%+ | **自治委任系统** — 可将某类决策交给 AI |
| 委员会制度建立 | 财务/技术/教育委员会各自负责对应领域 |
| 融合循环 Lv.4+ | 正向循环完全自持 |

### 终局：退休（Q12+）

当同时满足以下条件时，触发「退休」选项：

| 条件 | 阈值 |
|---|---|
| 现金储备 | ≥ ¥300,000 |
| 融合度 | Lv.5 |
| 制度成熟度 | 各维度 ≥ 80% |
| 团队 | ≥ 10 人，含 3 名以上团队负责人 |
| 升级 | 自动化满级，平台 ≥ Lv.3 |
| 连续盈利 | 连续 4 季度净利润为正 |

退休后游戏**不结束**，切换到观察者模式，看公司在你留下的制度下继续运转。

## 三、功能需求

### 3.1 P0 — 核心循环（MVP 已实现）

| 功能 | 说明 | 状态 |
|---|---|---|
| 项目系统 | 数据/教育类项目刷新、承接、执行、完成 | ✅ 已有 |
| 资金系统 | 项目收入、升级支出、工资支出 | ✅ 已有 |
| 团队系统 | 成员能力值、招聘 | ✅ 已有 |
| 升级系统 | 5 条投资线 | ✅ 已有 |
| 产教融合循环 | 双知识→融合度→效率加成 | ✅ 已有 |
| 随机事件 | 带多选项的事件弹窗 | ✅ 已有 |
| 季度结算 | 评级、评分、下一季度 | ✅ 已有 |

### 3.2 P1 — 制度成熟度（新增核心系统）

**这是"退休也能运转"的根基。**

#### 三维度

| 维度 | 积累方式 | 上限 | 效果 |
|---|---|---|---|
| 📋 标准化率 | 每完成 3 个项目 +1，投资"流程自动化"额外加 | 100 | 每 20 点 → 项目效率 +5% |
| 🤝 去中心化度 | 建立委员会、委派决策、团队成员自治决策 | 100 | 每 20 点 → 可多委派一种决策类型 |
| 📖 透明化度 | 开放数据给团队、召开代表大会、公示经营数据 | 100 | 每 20 点 → 团队满意度 + 事件正面率提升 |

#### 制度里程碑

| 里程碑 | 触发条件 | 效果 |
|---|---|---|
| 公司代表大会 | 去中心化度 ≥ 30 | 解锁**集体决策**：某些重大事件由委员会投票而非玩家独断 |
| 财务委员会 | 透明化度 ≥ 40 | 财务管理自动化：现金流预警自动处理 |
| 技术委员会 | 标准化率 ≥ 50 | 技术选型、工具采购由委员会负责 |
| 教育委员会 | 去中心化度 ≥ 50 + 教育知识 ≥ 60 | 课程体系维护、实习生管理自动运转 |
| 退休条款 | 三维度均 ≥ 80 | 解锁退休按钮 |

### 3.3 P1 — 自治委任系统

玩家可以将特定决策领域委任给对应的委员会，委任后该领域的事件将由系统自动选择最优解，不再弹窗打扰玩家。

| 委任领域 | 前置要求 | 委任后行为 |
|---|---|---|
| 项目选择 | 标准化率 ≥ 40 | 自动承接最优项目（基于当前资源配比） |
| 团队管理 | 去中心化度 ≥ 30 | 自动招聘/辞退，保持团队规模与技能平衡 |
| 财务投资 | 财务委员会已建立 | 自动执行升级购买（资金充裕时） |
| 全部委任 | 三委员会均已建立 | 完全自治模式——玩家成为观察者 |

### 3.4 P2 — 资产负载表与深度财务

| 需求 | 说明 |
|---|---|
| 利润表 | 季度收入、成本、利润明细 |
| 资产负债表 | 现金、应收账款、固定资产（升级）、人力资产 |
| 现金流预测 | 未来 1-2 季度的资金预测曲线 |
| 财务比率 | 人效比、毛利率、经营杠杆率 |

### 3.5 P2 — 市场与竞争

| 需求 | 说明 |
|---|---|
| 市场热度波动 | 不同季度/年份，数据/教育市场需求周期性变化 |
| 竞争对手出现 | 随机出现竞对，影响项目报价和成功率 |
| 行业趋势事件 | 如"AI 浪潮""教育改革政策"等，影响整个市场格局 |

### 3.6 P2 — 人才深度系统

| 需求 | 说明 |
|---|---|
| 成员成长 | 参与者执行匹配项目，能力值逐步提升 |
| 成员特质 | 每个人有性格标签（稳健型/激进型/技术型/管理型），影响自动决策倾向 |
| 团队文化 | 团队整体文化倾向影响去中心化进程 |

### 3.7 P3 — 多周目

退休后，玩家可以开启新游戏+：新公司继承部分制度点，初始效率加成，但面对不同的市场环境（随机种子变化）。

## 四、退休系统详解

### 4.1 触发退休

玩家**主动点击**退休按钮（非强制），满足条件才可点击。退休是成就，不是惩罚。

### 4.2 退休后模式

退休后页面从「指挥所」切换为「纪念馆」：

- 上方显示：退休年份、经营总季度数、公司最终规模
- 中央区域：实时播放公司经营快照的**时序回放**（各维度指标随时间变化的动画）
- 下方：**续行模拟器** — 系统自动推进季度，玩家观察公司按既有制度运转
- 右下角：随时可以"回归掌舵"重新接管

### 4.3 退休评分

| 评分维度 | 权重 | 计算方式 |
|---|---|---|
| 制度建设 | 30% | 退休时三维度平均值 |
| 财务健康 | 25% | 现金 + 连续盈利季度数 |
| 团队规模 | 15% | 人数 + 团队负责人数量 |
| 市场口碑 | 15% | 信誉值 + 客户总数 |
| 产教融合 | 15% | 融合度 Lv.5 + 双知识总量 |

总分 S/A/B/C 四档，影响多周目继承加成。

### 4.4 续行模拟（退休后的世界）

模拟器自动运行时，每个季度快进 2 秒展示关键指标变化：

- 项目自动承接、执行、完成
- 自动升级（资金充裕时）
- 自动招聘/解聘
- 随机事件出现时，由委员会自动决策

玩家随时可以暂停、回退到退休前的某个节点、或者重新接管。

**续行 8 季度后**，生成一份《遗产报告》：总结公司在你退休后的表现，对比你掌舵时的数据。

## 七、边界情况

| 场景 | 处理 |
|---|---|
| 退休后公司破产 | 续行模拟中财务归零 → 触发《破产报告》，对比玩家掌舵时的表现 |
| 退休条件部分满足 | UI 上显示各条件的达成进度，提示"还差什么" |
| 玩家回归接管 | 模拟中断，恢复到正常经营模式，所有委任保留 |
| 多周目存档覆盖 | 保留前一周目的最高评分记录，可查看历史 |
